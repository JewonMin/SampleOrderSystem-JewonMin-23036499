# 시스템 개요

## 배경

반도체 회사 **S-Semi**는 시료 주문량 급증으로 인해 주문 추적, 재고 파악, 공정 중복 등의
문제가 발생하여 **반도체 시료 생산주문관리 시스템**을 개발한다.

---

## 등장 역할

| 역할 | 설명 |
|------|------|
| **주문 담당자** | 주문서 작성 및 관리 |
| **생산 담당자** | 시료 등록, 주문 승인/거절, 생산 처리 |

---

## 시스템 특성

- **콘솔 기반** 동작 — 담당자가 직접 명령 입력
- **생산 라인** 단위로 시료를 하나씩 생산
- 주문이 들어온 시료에 대해서만 생산 진행

---

## 주문 상태 흐름도

```
[주문 담당자]  주문 등록 → RESERVED
                               │
[생산 담당자]            승인 여부
                        ┌──────┴──────┐
                      거절           승인
                        │               │
                     REJECTED       재고 확인
                                ┌──────┴──────┐
                             재고 충분     재고 부족
                                │               │
                            CONFIRMED       PRODUCING
                                │           (생산 완료)
                                └──────┬──────┘
                                    RELEASED
```

### 주문 상태 정의

| 상태 | 설명 |
|------|------|
| `RESERVED` | 주문 등록 완료, 승인 대기 |
| `REJECTED` | 생산 담당자 거절 |
| `CONFIRMED` | 재고 충분, 출고 준비 완료 |
| `PRODUCING` | 재고 부족, 생산 라인 투입 |
| `RELEASED` | 출고 완료 |
