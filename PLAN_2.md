## 2. 모호한 항목 (검토 필요)

### 2-A. 모니터링 재고 상태 — "주문 대비" 기준이 무엇인가?

PRD: "여유 = 주문 대비 재고 충분", "부족 = 주문 대비 재고 수량 부족"

"주문"의 범위가 명확하지 않음. 아래 두 해석 중 선택 필요:

| 해석 | 기준 주문 | 비고 |
|------|-----------|------|
| **A안** | RESERVED + PRODUCING 주문의 해당 시료 수량 합산 | 아직 처리되지 않은 수요 기준 |
| **B안** | 단순히 재고 0 여부만으로 고갈/비고갈 판단 | 잔여율 = stock 절대값 기반 |

→ **A안 추천**: 해당 시료의 미처리 주문(RESERVED + PRODUCING) 수량 합 vs 현재 재고 비교.
잔여율 = `stock / (stock + 미처리주문수량)` 또는 설계 편의에 따라 결정.

---

### 2-B. 모니터링 재고 표시 — 진행 중 생산량 통합 vs 분리

`docs/features/production-line.md`:
"생산이 진행되면서 생산된 수량은 **평균 생산시간에 비례하여 실시간으로 현재 재고에 반영**됨"

- 현재 구현: 승인 판단 시 `effectiveStock()` 사용, 모니터링 재고는 `sample.stock` + `inProductionQty` 분리 표시.
- 모니터링에서 진행 중 생산량을 재고에 통합해서 보여야 하는가?

→ **현재 방식(분리 표시) 유지 추천**: inProductionQty가 분리로 표시되어 정보량이 더 높음.
→ 단, 재고 상태 분류(여유/부족) 판단 시에는 `effectiveStock()` 사용 여부 결정 필요.

---

### 2-C. 생산라인 — FIFO 순서 엄격 적용 여부

FIFO 단일 라인 명세상 실제로 "처리 중"인 항목은 1개뿐이지만,
현재 `completeProduction()`은 큐 내 임의 항목 완료를 허용.

→ 첫 번째 항목(현재 처리 중인 항목)만 완료 가능하도록 제한할 것인지 결정 필요.

| 오류 메시지 | Red Bold |

| 구분선·헤더 | 기본 또는 White Bold |
